Swift URL Routing is a library for building composable, type-safe URL routers that work bidirectionally — parsing incoming URLs into well-structured Swift values, and printing those values back into URLs. It works on both client-side iOS apps (for deep linking) and server-side Swift apps.

## Why Swift URL Routing?

Traditional URL routers only go one direction — they match incoming URLs but leave you to manually construct outgoing URLs. This library solves both problems at once.

<CardGroup cols={2}>
  <Card title="Bidirectional" icon="arrows-left-right">
    The same router that parses `/books/42` can generate `/books/42` from a `.book(id: 42)` route value. No duplication.
  </Card>
  <Card title="Type-safe" icon="shield-check">
    Routes are modeled as Swift enums with associated values. The compiler catches missing cases and type mismatches.
  </Card>
  <Card title="Composable" icon="puzzle-piece">
    Build routers from small, reusable pieces using result builder syntax for paths, queries, headers, and bodies.
  </Card>
  <Card title="Testable" icon="flask">
    `URLRoutingClient` ships with a `failing` client and `override` API for unit testing without real network calls.
  </Card>
</CardGroup>

## How it works

<Steps>
  <Step title="Model your routes as a Swift enum">
    Define an enum where each case represents a URL in your app. Associated values hold the data extracted from the URL.

    ```swift
    enum AppRoute {
      case books
      case book(id: Int)
      case searchBooks(query: String, count: Int = 10)
    }
    ```
  </Step>
  <Step title="Build a router with composable parsers">
    Describe how each route maps to URL components using `Route`, `Path`, `Query`, and other parsers.

    ```swift
    import URLRouting

    let appRouter = OneOf {
      Route(.case(AppRoute.books)) {
        Path { "books" }
      }
      Route(.case(AppRoute.book(id:))) {
        Path { "books"; Digits() }
      }
      Route(.case(AppRoute.searchBooks(query:count:))) {
        Path { "books"; "search" }
        Query {
          Field("query")
          Field("count", default: 10) { Digits() }
        }
      }
    }
    ```
  </Step>
  <Step title="Parse URLs and generate them">
    Use `match(url:)` to parse incoming URLs, and `url(for:)` or `path(for:)` to generate URLs from route values.

    ```swift
    // Parsing
    try appRouter.match(url: URL(string: "/books/42")!)
    // → AppRoute.book(id: 42)

    // Printing
    appRouter.path(for: .searchBooks(query: "Blob Bio"))
    // → "/books/search?query=Blob%20Bio"
    ```
  </Step>
</Steps>
