This guide walks you through defining routes, building a router, and using it to both parse and generate URLs.

## Prerequisites

Make sure you've added URLRouting to your project. See [Installation](/installation) for details.

## Build your first router

<Steps>
  <Step title="Import URLRouting">
    Add the import at the top of your Swift file:

    ```swift
    import URLRouting
    ```

    This also re-exports the `swift-parsing` library, so all parser types (`OneOf`, `Digits`, `Int.parser()`, etc.) are available immediately.
  </Step>

  <Step title="Define your routes as an enum">
    Model each URL in your app as an enum case. Put only the data you want to extract into the associated values — not the URL structure itself.

    ```swift
    enum AppRoute {
      case books
      case book(id: Int)
      case searchBooks(query: String, count: Int = 10)
    }
    ```

    This enum represents three endpoints:
    - `GET /books` — list all books
    - `GET /books/:id` — show a specific book
    - `GET /books/search?query=:query&count=:count` — search books
  </Step>

  <Step title="Build the router">
    Use `OneOf` to combine individual `Route` parsers, one per enum case. Each `Route` describes where in the URL the data lives.

    ```swift
    let appRouter = OneOf {
      // GET /books
      Route(.case(AppRoute.books)) {
        Path { "books" }
      }

      // GET /books/:id
      Route(.case(AppRoute.book(id:))) {
        Path { "books"; Digits() }
      }

      // GET /books/search?query=:query&count=:count
      Route(.case(AppRoute.searchBooks(query:count:))) {
        Path { "books"; "search" }
        Query {
          Field("query")
          Field("count", default: 10) { Digits() }
        }
      }
    }
    ```

    - `Path { "books"; Digits() }` matches the path `/books/42` and extracts `42` as an `Int`.
    - `Query { Field("query"); Field("count", default: 10) { Digits() } }` reads `?query=...&count=...`.
    - `.case(AppRoute.book(id:))` is a `Conversion` from swift-parsing that maps the extracted values to the enum case.
  </Step>

  <Step title="Parse an incoming URL">
    Use `match(url:)` or `match(path:)` to turn a URL into an `AppRoute` value:

    ```swift
    func handleDeepLink(url: URL) throws {
      switch try appRouter.match(url: url) {
      case .books:
        // navigate to books screen

      case let .book(id: id):
        // navigate to book with id

      case let .searchBooks(query: query, count: count):
        // navigate to search screen with query and count
      }
    }
    ```

    ```swift
    // Parsing from a path string
    try appRouter.match(path: "/books/42")
    // → AppRoute.book(id: 42)

    try appRouter.match(path: "/books/search?query=Blob&count=5")
    // → AppRoute.searchBooks(query: "Blob", count: 5)

    // Default values are used when query params are absent
    try appRouter.match(path: "/books/search?query=Blob")
    // → AppRoute.searchBooks(query: "Blob", count: 10)
    ```
  </Step>

  <Step title="Generate URLs from route values">
    The same router can print route values back into URLs — no separate URL construction logic needed:

    ```swift
    appRouter.path(for: .books)
    // → "/books"

    appRouter.path(for: .book(id: 42))
    // → "/books/42"

    appRouter.path(for: .searchBooks(query: "Blob Bio"))
    // → "/books/search?query=Blob%20Bio"
    // (count is omitted because it equals the default value of 10)

    appRouter.path(for: .searchBooks(query: "Blob Bio", count: 5))
    // → "/books/search?query=Blob%20Bio&count=5"
    ```

    To generate absolute URLs, prepend a base URL:

    ```swift
    let apiRouter = appRouter.baseURL("https://api.example.com")

    apiRouter.url(for: .book(id: 42))
    // → URL("https://api.example.com/books/42")
    ```
  </Step>

  <Step title="Use URLRoutingClient for API requests">
    On iOS and macOS, use `URLRoutingClient` to make type-safe network requests:

    ```swift
    // Create a live client
    let apiClient = URLRoutingClient.live(
      router: appRouter.baseURL("https://api.example.com")
    )

    // Make a request and decode the JSON response
    struct Book: Decodable { var id: Int; var title: String }

    let (book, _) = try await apiClient.decodedResponse(
      for: .book(id: 42),
      as: Book.self
    )
    print(book.title) // "Blob Autobiography"
    ```

    For testing, use the `failing` client with `override` to mock responses without hitting the network:

    ```swift
    let testClient = URLRoutingClient<AppRoute>.failing
      .override(.books) {
        try .ok([Book(id: 1, title: "Blob Autobiography")])
      }
    ```
  </Step>
</Steps>
