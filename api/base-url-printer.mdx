---
title: "BaseURLPrinter"
description: "Wraps a router to prepend base URL data when printing. Does not affect parsing."
---

## Overview

`BaseURLPrinter<Upstream>` lets you attach a base URL (or any default `URLRequestData`) to a router so that printed routes produce absolute URLs rather than relative paths.

You will rarely construct `BaseURLPrinter` directly. Use the `baseURL(_:)` or `baseRequestData(_:)` extension methods on any `ParserPrinter` whose `Input` is `URLRequestData`.

Parsing is unaffected â€” the base data is only applied during printing.

## Declaration

```swift
public struct BaseURLPrinter<Upstream: ParserPrinter>: ParserPrinter
where Upstream.Input == URLRequestData
```

## Creation

Both methods are available as extensions on `ParserPrinter where Input == URLRequestData`.

### `baseURL(_:)`

Prepends a base URL string when printing. Scheme, user, password, host, port, path, query, and fragment are all extracted from the string.

```swift
@inlinable
public func baseURL(_ urlString: String) -> BaseURLPrinter<Self>
```

**Parameters**

| Name | Type | Description |
|------|------|-------------|
| `urlString` | `String` | A valid URL string, e.g. `"https://api.example.com/v1"`. Calling this with an invalid URL string causes a `fatalError` at runtime. |

**Returns** A `BaseURLPrinter` that prepends the base URL to every printed route.

### `baseRequestData(_:)`

Prepends arbitrary default `URLRequestData` when printing. Use this when you need to inject headers, custom ports, or other fields that cannot be expressed as a plain URL string.

```swift
@inlinable
public func baseRequestData(_ requestData: URLRequestData) -> BaseURLPrinter<Self>
```

**Parameters**

| Name | Type | Description |
|------|------|-------------|
| `requestData` | `URLRequestData` | Default request data merged into every printed output. |

## Behavior when printing

After the upstream router prints a route into a `URLRequestData` value, `BaseURLPrinter` applies the following merge rules from the base data:

- **scheme** â€” overrides the printed scheme if set in base.
- **user** â€” overrides if set in base.
- **password** â€” overrides if set in base.
- **host** â€” overrides if set in base.
- **port** â€” overrides if set in base.
- **path** â€” base path components are *prepended* to the printed path.
- **query** â€” base query parameters are *merged* with printed query parameters; base values take lower priority (printed values win on collision).
- **fragment** â€” overrides if set in base.
- **headers** â€” base headers are *merged* with printed headers; base values take lower priority.

## Parsing

`BaseURLPrinter.parse(_:)` delegates directly to the upstream router. The base data is completely ignored during parsing, so a `BaseURLPrinter` can be used as a drop-in replacement for its upstream in any parsing context.

```swift
@inlinable
public func parse(_ input: inout URLRequestData) rethrows -> Upstream.Output
```

## Examples

### Generating absolute URLs with `baseURL`

```swift
enum AppRoute {
  case home
  case episodes
}

let router = OneOf {
  Route(AppRoute.home)
  Route(AppRoute.episodes) {
    Path { "episodes" }
  }
}

let apiRouter = router.baseURL("https://api.pointfree.co/v1")

// Printing produces absolute URLs:
let requestData = try apiRouter.print(.episodes)
URLRequest(data: requestData)?.url?.absoluteString
// "https://api.pointfree.co/v1/episodes"
```

### Including query parameters in the base URL

```swift
let apiRouter = router.baseURL("https://api.pointfree.co/v1?token=deadbeef")

URLRequest(data: try apiRouter.print(.episodes))?.url?.absoluteString
// "https://api.pointfree.co/v1/episodes?token=deadbeef"
```

### Adding default headers with `baseRequestData`

Use `baseRequestData` to inject headers (such as authentication tokens) into every request generated by the router.

```swift
let authenticatedRouter = router
  .baseRequestData(.init(headers: ["X-PointFree-Session": ["deadbeef"]]))

let request = try authenticatedRouter.request(for: .home)
request.value(forHTTPHeaderField: "x-pointfree-session")
// "deadbeef"
```

<Tip>
Combine `baseURL` and `baseRequestData` by chaining them, or supply a fully constructed `URLRequestData` to `baseRequestData` that includes both a host and custom headers.
</Tip>

<Warning>
Passing an invalid URL string to `baseURL(_:)` causes a `fatalError` at runtime. Validate URL strings before calling this method, or use `baseRequestData(_:)` with a manually constructed `URLRequestData` for safer initialization.
</Warning>
